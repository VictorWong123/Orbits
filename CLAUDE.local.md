## Rules

- Write modular code and add docstrings to all methods.
    -Each function should have its own function, break down larger functions into smaller ones. 
- Write unit tests when applicable (especially when something uses a lot of logic)
-use playwright_mcp and context7 as tools to help solve issues. Always run test cases and playwrite__mcp after you are done to ensure you don't break anything.
- After each task, document exactly what you did and any decision making. It should be highly technical.
- Each file/component should have one main function and private helper methods. If there are two main functions, split it up into two files. This is to keep code modular.
